Summary Tables

In the example above, we have a "summary" table, which is a table that summarizes content and allows the user to take an action to view all the details. In a summary table, there are some columns that are more important than others. On smaller viewports, it is acceptable to hide the unessential columns, as the user can take an action to view full details. We hide unnecessary columns by using .data-p[1-4] classes to indicate the importance of a specific column. The .data-p* classes are then hidden on smaller viewports. The exact breakpoint at which a specific .data-p* class may vary, based on the context of the table.

Example .data-p* classes

.data-p1 {
    display:table-cell;
}

.data-p2,
.data-p3,
.data-p4 {
    display:none;
}

@include bp (min-width, 480px) {

    .data-p2 {
        display:table-cell;
    }

}

@include bp (min-width, 600px) {

    .data-p3 {
        display:table-cell;
    }

}

@include bp (min-width, 768px) {

    .data-p4 {
        display:table-cell;
    }

}
Table - Linearize

Tables with many columns do not display well on small viewports. While hiding columns on *summary* tables is acceptable, it is not acceptable on data tables because users we should not restrict access to information based on viewport size. In order to display all of the data in this type of table without forcing the user to scroll, we use a "linearize" pattern that turns cells into rows.

The markup for a linearized table follows this pattern

<table class="linearize-table">
    <thead>
        <tr>
            <th>Product Name</th>
            <th>Price</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <-- The data-label attribute will be added via JS -->
            <td data-label="Product Name">Mountain Bike</td>
            <td data-label="Price">$3,000</td>
        </tr>
        <tr>
            <td data-label="Product Name">Helment</td>
            <td data-label="Price">$180</td>
        </tr>
    </tbody>
</table>

When a linearize table is displayed on a small viewport, the "data-label" attribute is used as the label for that cell 

.linearize-table tbody td[data-label]:before {
    content:attr(data-label) ":";
}

Rather than adding the data-label attribute to all tbody td elements, we have a jQuery decorator that does this for us

The decorator allows us to write linearize table markup without the data-label attributes. This is useful because Magento has multiple item render template files, and using a decorator eliminates the need to update label/th values in multiple templates.

In some linearized tables, the first column in the table does not need to be prepended with a label, as its meaning is apparent (eg, product name). For this type of column, a "data-no-label" attribute can be added to the th. You can see an example of this in the linearized table below. 

// ==============================================
// UI Pattern - Linearize Table
// ==============================================

$('.linearize-table').each(function(){

    var cellLabels = [];

    // Loop through all table headers to find the labels
    $(this).find('thead th').each(function(index){
        // Only assign label if there is no data-no-label attribute on the th. Adding data-no-label will display
        // the value of each cell without it being prepended by a label, which can be useful for product name and
        // similar data types.
        cellLabels[index] = ($(this).is('[data-no-label]')) ? false : $(this).text().trim();
    });

    $(this).find('tbody tr').each(function(){
        // Looping through each td inside of each row so that the index value will match that of thead > tr > th
        $(this).children('td').each(function(index){
            var label = cellLabels[index];
            // As long as the th did not contain a data-no-label attribute, add the data-label attribute
            if (label) {
                $(this).attr('data-label', label);
            }
        });
    });
});

Using the JS decorator, we can now write our markup like this

<table class="linearize-table">
    <thead>
        <tr>
            <th data-no-label>Product Name</th>
            <th>Price</th>
            <th>Qty</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Mountain Bike</td>
            <td>$3,000</td>
            <td>2</td>
        </tr>
        <tr>
            <td>Helment</td>
            <td>$180</td>
			<td>1</td>
        </tr>
    </tbody>
</table>
Table - Linearize Custom

For tables that are a more prominent part of the user experience (eg, cart items table), using a vanilla linearize pattern is not appropriate, as the screen real estate can be used in a more effective manner. For an example of this, reference the cart on the shop.angrybirds.com site.
